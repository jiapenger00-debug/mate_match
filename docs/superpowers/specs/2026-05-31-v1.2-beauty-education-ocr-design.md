## 二、截图解析

### 概述

用户上传社交主页/交友资料截图，多模态 LLM 识别提取关键信息，自动填入表单。

### 交互流程

```
┌─ 表单顶部 ──────────────────────────────┐
│  [📷 截图导入]  或直接 Ctrl+V 粘贴图片    │
└─────────────────────────────────────────┘
        │
        ▼
┌─ 预览弹窗 ──────────────────────────────┐
│  [图片预览]                              │
│  "正在识别中…"                           │
└─────────────────────────────────────────┘
        │ POST /api/ocr (multipart/form-data)
        ▼
┌─ vision_service.ocr_extract(image) ─────┐
│  多模态 LLM：                             │
│  "提取姓名、年龄、职业、学校、            │
│   性格特征、兴趣爱好… 返回 JSON"          │
└─────────────────────────────────────────┘
        │ 返回 { name, age, occupation, ... }
        ▼
   自动填入表单字段（高亮已填充的字段）
```

### 后端

新增路由：`POST /api/ocr`

- 接收 `multipart/form-data`，字段 `image`（图片文件）
- 调用 `vision_service.ocr_extract(image_bytes)`
- 返回 JSON：`{"name": "林小曼", "age": 25, "occupation": "设计师", ...}`
- 空字段返回 `null`

### vision_service.ocr_extract()

```python
async def ocr_extract(image_bytes: bytes, backend: VisionBackend) -> dict:
    prompt = """分析这张截图，提取以下信息，返回 JSON：
    { "name": "姓名", "age": 年龄数字, "hometown": "家乡城市",
      "occupation": "职业", "education": "学历或学校",
      "personality": "性格描述", "interests": "兴趣爱好",
      "values": "价值观或人生追求", "appearance": "外貌特征" }
    未提及的字段返回 null，不要猜测。"""
    return await backend.analyze(image_bytes, prompt)
```

### 前端

- `index.html` 顶部（女方和用户卡片上方）各加一个"截图导入"按钮
- 全局 `paste` 事件监听：检测剪贴板中的图片，直接触发识别
- 识别完成后逐个填入字段，填入时闪烁绿色边框提示

### 文件变化

| 文件 | 操作 |
|------|------|
| `services/vision_service.py` | 新建，ocr_extract() |
| `main.py` | 新增 POST /api/ocr |
| `templates/index.html` | 截图按钮 + paste 监听 |
| `static/style.css` | 预览弹窗 + 字段高亮动画 |

## 三、颜值评分

### 概述

支持上传双方照片，AI 进行面部吸引力评分，结果纳入匹配分析。提供表单内嵌上传 + 独立 PK 页两种入口。

### 表单内嵌模式

在女方/用户卡片中各加一个上传区（拖拽/点击），选填：

```
┌─ 上传区 ───────────────────┐
│    📷                       │
│  点击或拖拽上传照片          │
│  (选填，用于颜值分析)        │
│  [已上传: photo.jpg ✓]     │
└────────────────────────────┘
```

提交分析时把照片一起 POST 到 `/api/analyze`（multipart 支持）。

### 独立 PK 页 `GET /beauty`

新页面，双方照片并排 + 颜值打分 + 匹配度：

```
┌──────────────┐  VS  ┌──────────────┐
│   👩 女方     │      │   🧑 男方     │
│  [上传照片]   │      │  [上传照片]   │
│              │      │              │
│  颜值分: 8.5  │      │  颜值分: 7.2  │
│  肤质: 细腻   │      │  肤质: 匀称   │
│  脸型: 鹅蛋脸 │      │  脸型: 方脸    │
│              │      │              │
│  颜值匹配度: 82 分 很登对！          │
└────────────────────────────────────┘
```

### 后端

新增路由：`POST /api/beauty`

- 接收 `multipart/form-data`，字段 `girl_photo` + `user_photo`（图片文件）
- 分别调用 `vision_service.analyze_beauty()` 对两张照片评分
- 调用 `llm_service` 生成颜值匹配分析
- 返回 JSON：`{ girl: BeautyResult, user: BeautyResult, match_score, match_comment }`

### vision_service.analyze_beauty()

```python
async def analyze_beauty(image_bytes: bytes, backend: VisionBackend) -> BeautyResult:
    prompt = """分析这张人脸照片，进行面部吸引力评估。返回 JSON：
    {
      "score": 1-10 的颜值分数（保留一位小数，5 为平均），
      "skin": "肤质评价（细腻/光滑/匀称/一般/粗糙）",
      "face_shape": "脸型（鹅蛋脸/瓜子脸/圆脸/方脸/长脸/心形脸）",
      "features": "五官特点简要描述",
      "overall": "整体印象评价，20 字以内"
    }
    客观评价，不要刻意夸张。"""
    return await backend.analyze(image_bytes, prompt)
```

### 汇总到匹配分析

在 `/api/analyze` 中，如果用户上传了照片，颜值结果作为新的维度加入 prompt：

```
## 颜值分析（基于照片 AI 评估）
- 女方：颜值 8.5 分，鹅蛋脸，肤质细腻，五官精致
- 男方：颜值 7.2 分，方脸，肤质匀称，阳光气质
- 颜值匹配度：82 分
```

## 四、视觉服务架构

### 双后端设计

```
vision_service.py
├── VisionBackend (ABC)
│   ├── analyze(image_bytes, prompt) -> dict
│   └── supports_ocr() -> bool
│
├── QwenBackend(VisionBackend)       # VISION_BACKEND=qwen_api
│   ├── 调用阿里云 DashScope API
│   ├── 模型: qwen-vl-max
│   └── 支持 OCR + 颜值
│
└── OllamaBackend(VisionBackend)     # VISION_BACKEND=local_ollama
    ├── 调用本地 Ollama HTTP API
    ├── 模型: qwen2-vl:7b
    └── 支持 OCR + 颜值
```

### 配置

```env
# .env 新增

# 视觉后端选择
VISION_BACKEND=qwen_api

# Qwen API（VISION_BACKEND=qwen_api 时必填）
QWEN_API_KEY=sk-xxx

# Ollama（VISION_BACKEND=local_ollama 时使用）
OLLAMA_MODEL=qwen2-vl:7b
OLLAMA_BASE_URL=http://localhost:11434
```

### 后端选择逻辑

```python
# services/vision_service.py
from config import VISION_BACKEND

def get_backend() -> VisionBackend:
    if VISION_BACKEND == "local_ollama":
        return OllamaBackend()
    elif VISION_BACKEND == "qwen_api" or VISION_BACKEND == "":
        return QwenBackend()
    else:
        raise ValueError(f"Unknown VISION_BACKEND: {VISION_BACKEND}")
```

### Ollama 部署指南

```bash
# 安装 Ollama (https://ollama.com)
# 拉取模型（约 4GB）
ollama pull qwen2-vl:7b

# 启动服务（默认 localhost:11434）
ollama serve
```
