## 一、丰富示例库

### 概述

"一键填充示例"从固定一组升级为**标签栏切换 + 预览确认**模式，支持 4 组不同风格的情侣案例。

### 数据层

新建 `static/demo-data.js`，每个示例包含完整 girl/user 数据 + 元信息：

```js
const DEMO_EXAMPLES = [
  {
    id: "art-x-logic",
    label: "文艺 × 理工",
    emoji: "🎨",
    girl: { name: "林小曼", age: 25, ... },
    user: { name: "张浩然", age: 28, ... }
  },
  { id: "lively-x-calm", label: "活泼 × 沉稳", emoji: "🔥", ... },
  { id: "elite-x-free",  label: "精英 × 自由", emoji: "👔", ... },
  { id: "dev-x-dev",     label: "码农 × 码农", emoji: "💻", ... },
];
```

示例数据手动编写，每条 200-400 字，覆盖不同性格组合。

### 前端交互

```
搜索开关
────────────────────────────
[🎨 文艺×理工] [🔥 活泼×沉稳] [👔 精英×自由] [💻 码农×码农]
────────────────────────────
┌─ 预览区 ─────────────────┐
│ 林小曼：开朗乐观，喜欢探店… │
│ 张浩然：稳重理性，喜欢骑行… │
└──────────────────────────┘
[取消填充]            [✅ 确认填充]
```

- 点击标签 → 预览区实时更新，当前标签高亮（边框 + 背景色变化）
- "确认填充" → 批量填入表单字段，标签变绿
- "取消填充" / 点击已选中标签 → 清空预览，取消填充
- `index.html` 在 `<head>` 中引用 `demo-data.js`

## 二、分析结果分享

### 概述

分析完成后生成唯一链接，支持分享。分享页为两层：精简名片 + 可展开完整报告。

### 后端

**存储**：SQLite，数据库文件 `data/shares.db`

```sql
CREATE TABLE IF NOT EXISTS shares (
    id           TEXT PRIMARY KEY,   -- 6 位短码
    girl_name    TEXT,
    user_name    TEXT,
    overall_score INTEGER,
    dimensions   TEXT,               -- JSON
    summary      TEXT,
    suggestion   TEXT,
    search_results TEXT,             -- JSON, 可为空
    created_at   TEXT DEFAULT (datetime('now'))
);
```

短 ID 生成：`secrets.token_urlsafe(4)` → 去掉 `-_` → 取 6 位。

**新增路由**：

| 路由 | 方法 | 职责 |
|------|------|------|
| `/api/share` | POST | 接收 JSON body（AnalyzeResponse 序列化），写入 SQLite，返回 `{ "id": "a3Xk9m" }` |
| `/s/<share_id>` | GET | 查询 SQLite，找不到返回 404 页，找到则渲染分享页 |

**新建** `services/share_service.py`：`save_share(data) -> str` / `get_share(id) -> dict | None`。

### 前端

**result.html** 底部新增分享栏：

```
[📋 复制链接]   [📱 二维码]
```

- "复制链接" → `navigator.clipboard.writeText(url)` + toast 提示
- "二维码" → 纯 CSS 生成（`canvas` 画二维码），弹出小弹窗

**分享页 `/s/<id>`** — `templates/share.html`（新建）：

```
精简名片模式（默认）：
┌─────────────────────────────┐
│        💘 灵魂契合度          │
│     林小曼 × 张浩然          │
│      ┌───────┐              │
│      │  78   │              │
│      │ 契合度 │              │
│      └───────┘              │
│    心有灵犀                   │
│  "两人总体匹配度较高..."      │
│  [查看完整报告] [我也测测]    │
└─────────────────────────────┘
```

点击「查看完整报告」→ 展开维度条 + 评语 + 建议（用 CSS `max-height` 过渡展开，无额外请求）  
「我也测测」→ 跳转 `/`

分享页复用现有 CSS 变量，支持暗夜/日暖主题切换。

### 文件变化

| 文件 | 操作 |
|------|------|
| `services/share_service.py` | 新建 |
| `main.py` | 新增 `/api/share` + `/s/<share_id>` |
| `templates/share.html` | 新建 |
| `templates/result.html` | 底部加分享栏 |
| `static/style.css` | 新增 `.share-bar` `.share-card` `.qrcode-modal` 等 |

## 三、搜索体验增强

### 概述

搜索结果从"默默无反馈"升级为三段式：搜索动画 → 结果卡片 → AI 分析。

### 交互流程

```
提交表单
  │
  ▼
POST /api/search  ← 新增路由，只搜不分析
  │
  ▼
┌─ 搜索动画覆盖层 ───────────┐
│  💫 正在搜索她的公开信息…   │
│  ████████░░░░░░░░  50%    │  ← 伪进度动画
│  已找到 3 条相关信息…       │
└────────────────────────────┘
  │ (收到结果后淡出)
  ▼
┌─ 结果卡片列表 ─────────────┐
│ ☑ 林小曼 ｜ 设计师作品集    │
│ ☑ 林小曼 ｜ 站酷主页       │
│ ☐ 林小曼的微博主页         │  ← 不相关可取消
│ [跳过，直接分析] [✅ 确认]  │
└────────────────────────────┘
  │
  ▼
POST /api/analyze  ← 带上筛选结果
  │
  ▼
┌─ AI 分析动画 ──────────────┐
│  🤖 正在解读两颗心的距离…   │
└────────────────────────────┘
  │
  ▼
result.html
```

### 后端

**新增路由**：

| 路由 | 方法 | 职责 |
|------|------|------|
| `/api/search` | POST | 接收表单数据，执行搜索，返回 `{ "results": [...], "token": "xxx" }` |

返回的 `token` 是搜索结果的临时哈希，用于幂等校验（可选，防止重复搜索）。

**修改路由**：

`/api/analyze` 新增可选参数 `selected_results`（JSON 字符串），如果传了就跳过搜索直接分析。

### 前端

`index.html` 新增三个 DOM 块：
- `.search-overlay` — 搜索动画覆盖层（全屏半透明蒙版 + 进度条）
- `.search-results` — 结果卡片列表
- `.search-actions` — 确认/跳过按钮

JS 改为异步三段式：`submit → fetch /api/search → show cards → user confirm → fetch /api/analyze → redirect`。

### 文件变化

| 文件 | 操作 |
|------|------|
| `main.py` | 新增 `/api/search`，改 `/api/analyze` 支持 `selected_results` |
| `templates/index.html` | 新增搜索覆盖层 + 结果卡片 HTML + JS 改写 |
| `static/style.css` | 新增 `.search-overlay` `.search-card` `.search-progress` |
